JSON Templates

We had a need to know when a 3rd party would change their JSON response that we consume. So I created a set of PL/SQL pacakges and tables to compare a "perfect" version that we expected to all incoming JSON responses. This tool allowed for different types of alerting and was very configurable for any JSON (Not just our specific JSON responses)

Plug-ins

Using Oracle APEX 22+ I created 5 Plug-ins to enhance our projects and standardize some UI elements in Oracle's RedWood theme.

  • Checkbox Operations
    • The main reason for this plug-in was to add checkboxes to a Classic Report in APEX. It needed to be standardize across all apps and allows very little developer interaction / coding to implement.
  • Sticky Headers
    • Classic reports in RedWood theme do not have headers that scroll with the user, causing the user to lose track of what a column is conveying. So I wrote this Dynamic Action plug-in to lock header rows on a classic report.
  • RDS Connection
    • Region Display Selectors in APEX are a powerful UI tool, but we needed some extra functionality. Specifically when you have other items / elements outside of a region (page header, footer, etc..) that are dependent on a specific region this plug-in allowed developers to hide and show any element based on which Region is being shown from the RDS.
  • Column Selector
    • With the RedWood theme APEX classic reports are the standard views. As we know, sometimes reports are not always just 2 or 3 columns they can be 10 to 20 or even more. So with that in mind I created this plug-in that will hide and show columns on a classic report.
  • Navigation Expander
    • With the new idea of "Micro Apps" in APEX there is a need to track what state a user had left the navigation bar in (Standard APEX expandable navigation bar). For example if a user jumps to a new application the navigation bar may close even though they had it open. So this plug-in makes sure the navigation bar stays expanded or closed as the user jumps between applications.
